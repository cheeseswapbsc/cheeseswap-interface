## Summary

The refresh button implementation provides:

1. **Clear user feedback** - Users know when connection fails
2. **Manual control** - Users decide when to try fallback
3. **Backward compatibility** - Works with existing code
4. **Flexible fallback** - Generic injected works with any wallet
5. **Smart detection** - Specific connectors preferred when available

The flow ensures that:
- Specific wallets are tried first (best UX)
- Failures are communicated clearly
- Users have control over fallback
- In-app browsers work correctly
- No silent failures or confusion
